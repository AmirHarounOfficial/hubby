GitHub to aapanel CI/CD deployment guide with step-by-step phases

GitHub repo
→
GitHub Actions CI
→
SSH deploy
→
aapanel server
1
Set up GitHub repository
one-time

Create a new private repo on GitHub (e.g. hubbyglobal). Keep it private since it contains infra config.

Organise your project with a monorepo structure:

hubbyglobal/ ├── backend/ ← Laravel app ├── frontend/ ← Next.js / Vue / React ├── docker/ │ └── php.Dockerfile └── docker-compose.yml ← your file goes here

Add a .gitignore that excludes .env, node_modules/, vendor/, and storage/logs/*.

Push your initial code:

git init git remote add origin git@github.com:you/hubbyglobal.git git add . git commit -m "chore: initial commit" git push -u origin main
Never commit .env files. All secrets go in GitHub → Settings → Secrets.
2
Prepare aapanel server
one-time

In aapanel, create a dedicated deploy user (or use your existing SSH user). Make sure Docker and Docker Compose are installed:

curl -fsSL https://get.docker.com | sh apt install docker-compose-plugin -y

Create a deploy key pair for GitHub Actions to use:

ssh-keygen -t ed25519 -C "github-actions-deploy" -f ~/.ssh/deploy_key # No passphrase cat ~/.ssh/deploy_key.pub >> ~/.ssh/authorized_keys

Create your app directory and .env file on the server (this is the production env — not committed to git):

mkdir -p /www/wwwroot/hubbyglobal nano /www/wwwroot/hubbyglobal/.env # Fill in your production DB, Redis, APP_KEY values

In aapanel, create a site pointing to /www/wwwroot/hubbyglobal/backend/public and configure SSL via Let's Encrypt.

The frontend port 3000 and API port 8000 should be proxied through aapanel's reverse proxy — not exposed directly.
3
Add secrets to GitHub
one-time

Go to your GitHub repo → Settings → Secrets and variables → Actions. Add these secrets:

Secret name Value
SSH_PRIVATE_KEYContents of ~/.ssh/deploy_key (private key)
SERVER_HOSTYour server IP or domain
SERVER_USERSSH username (e.g. root or deploy)
SERVER_PORT22 (or your custom SSH port)
4
GitHub Actions CI/CD workflow
automated

Create the file .github/workflows/deploy.yml in your repo:

name: CI on: pull_request: branches: [main] push: branches: [main] jobs: test-backend: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: shivammathur/setup-php@v2 with: php-version: '8.2' extensions: pdo, mbstring, tokenizer - run: cd backend && composer install --no-interaction - run: cd backend && cp .env.testing .env - run: cd backend && php artisan key:generate - run: cd backend && php artisan test test-frontend: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 with: node-version: '20' cache: 'npm' cache-dependency-path: frontend/package-lock.json - run: cd frontend && npm ci - run: cd frontend && npm run lint - run: cd frontend && npm run build
deploy: needs: [test-backend, test-frontend] runs-on: ubuntu-latest if: github.ref == 'refs/heads/main' steps: - name: Deploy via SSH uses: appleboy/ssh-action@v1.0.3 with: host: ${{ secrets.SERVER_HOST }} username: ${{ secrets.SERVER_USER }} key: ${{ secrets.SSH_PRIVATE_KEY }} port: ${{ secrets.SERVER_PORT }} script: | cd /www/wwwroot/hubbyglobal git pull origin main docker compose -f docker-compose.yml pull docker compose up -d --build --remove-orphans docker compose exec -T app php artisan migrate --force docker compose exec -T app php artisan config:cache docker compose exec -T app php artisan route:cache docker compose exec -T app php artisan view:cache echo "Deploy complete ✓"
name: CI/CD Pipeline on: push: branches: [main] pull_request: branches: [main] jobs: test-backend: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: shivammathur/setup-php@v2 with: { php-version: '8.2' } - run: cd backend && composer install --no-interaction - run: cd backend && cp .env.testing .env && php artisan key:generate - run: cd backend && php artisan test test-frontend: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 with: { node-version: '20' } - run: cd frontend && npm ci && npm run lint && npm run build deploy: needs: [test-backend, test-frontend] runs-on: ubuntu-latest if: github.ref == 'refs/heads/main' steps: - uses: appleboy/ssh-action@v1.0.3 with: host: ${{ secrets.SERVER_HOST }} username: ${{ secrets.SERVER_USER }} key: ${{ secrets.SSH_PRIVATE_KEY }} port: ${{ secrets.SERVER_PORT }} script: | cd /www/wwwroot/hubbyglobal git pull origin main docker compose up -d --build --remove-orphans docker compose exec -T app php artisan migrate --force docker compose exec -T app php artisan optimize echo "Deploy complete ✓"
The deploy job only runs on pushes to main, not on PRs — so PRs only trigger tests.
5
Branch strategy & workflow
ongoing
feature branch
→
PR → tests run
→
merge to main
→
auto deploy

Work on feature branches: git checkout -b feat/my-feature

Open a PR — CI tests run automatically on every commit to the PR.

Merge to main only when tests pass — this triggers the deploy job automatically.

For hotfixes, create a hotfix/ branch, test, then merge to main.

Enable branch protection on main: require PR reviews + passing checks before merge (GitHub → Settings → Branches).
6
Zero-downtime & rollback tips
optional

Add a health check to your docker-compose.yml on the app service so Docker waits for it to be ready before routing traffic.

To roll back, SSH in and run git revert HEAD then push, or manually run git checkout <prev-sha> and re-run docker compose up -d --build.

Add a Slack/Discord notification step at the end of your workflow to alert on success or failure:

- name: Notify if: always() uses: slackapi/slack-github-action@v1.26.0 with: payload: | {"text": "Deploy ${{ job.status }} on ${{ github.repository }}"} env: SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK }}
